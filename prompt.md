넌 자본시장법 규제를 완벽히 준수하는 핀테크 웹 서비스 'IndexPlace AI'의 Senior Frontend 및 UI/UX 개발자이다.
제시된 기능 명세와 디자인 시스템을 바탕으로, PC 웹 브라우저 전체 화면을 사용하는 단일 파일 형태(Single HTML File: HTML/CSS/JavaScript 포함)의 와이드 뷰 웹 애플리케이션을 작성하라.

==================================================
1. 핵심 아키텍처 및 화면 레이아웃 (Navigation Structure)
==================================================
- 화면 상단에 서비스명('IndexPlace AI')과 규제 가드레일 상태 뱃지, 그리고 3개의 핵심 [독립 탭]을 상시 고정(Fixed Header) 배치한다.
- 사용자는 상단 탭을 통해 언제든 자유롭게 화면을 이동(Routing)할 수 있어야 한다.

- [화면 구성 4단계 게이트]:
  1) LOGIN SCREEN: 서비스 접속 전 전체 화면을 덮는 대형 로그인 카드 게이트. 로그인 성공 시 헤더 및 메인 워크스페이스가 활성화된다.
  2) TAB 1: [🔮 AI 챗봇 빌더] - 좌우 2분할(Split Screen) 레이아웃.
     - 좌측: 사용자 대화창 (Graph-RAG 인터페이스).
     - 우측: 대화 맥락에 따라 실시간으로 자산 배분 비중(%)이 계산되어 미리 보여지는 'ETF 미리보기 패널'.
  3) TAB 2: [🗺️ 밸류체인 온톨로지] - 독립된 대형 온톨로지 시각화 대시보드.
  4) TAB 3: [🛒 오픈 마켓플레이스] - 타 유저의 인덱스 지적재산권(IP)을 일회성 결제 형태로 사고파는 카드 그리드 레이아웃.

==================================================
2. 디자인 시스템 및 UI/UX 가이드라인 (Theme)
==================================================
- 전체 테마: 다크 딥 블루 / 다크 네이비 (#070a13 베이스, 카드 배경 #111827)
- 포인트 컬러: 
  - 기본 하이라이트/텍스트: 시안 블루 (#38bdf8)
  - 성공/MCP 완료: 엠버 그린 (#10b981)
  - 테마 노드: 퍼플 (#c084fc)
  - 역할 노드: 오렌지 (#fb923c)
  - 관계선 설명 라벨: 로즈 핑크 (#fb7185)
- 모던하고 차분한 프리미엄 핀테크 SaaS 스타일로 작성하며, 모든 모달과 카드엔 부드러운 라운딩(border-radius: 12px~20px)과 보더(#1f2937)를 적용하라.

==================================================
3. 3-Layer 금융 온톨로지 시각화 스키마 (Graph DB UI)
==================================================
TAB 2(온톨로지 대시보드) 화면은 다음 3단계 구조로 배치하고, 노드 간의 연결 관계선(Edge Path) 위에 공인 팩트 근거 텍스트 라벨(Edge Description)을 상시 노출하라.

- Layer 1 (왼쪽): [테마 노드] - 예: 온디바이스 AI (보라색 라운드 뱃지)
- Layer 2 (중앙): [역할군 노드] - 예: 칩 제조사, 검사 부품, IP 설계 (주황색 네모 노드)
- Layer 3 (오른쪽): [정합 상장 종목 노드] - 예: 삼성전자, 리노공업, ISC, 오픈엣지
  * 각 종목 노드 하단에는 시가총액 기반 체급 태그(대형주/중형주/소형주)가 서브 뱃지로 시각화되어야 함.
- 관계선 라벨(Edge Label):
  - 예: "금감원 DART: 초미세 검사 핀 독점 확인", "사업보고서: 메모리용 러버소켓 공급선" 등의 분홍색 텍스트 캡슐을 라인 중앙에 덮어 팩트 근거를 명시할 것.

==================================================
4. 핵심 기능 인터랙션 및 비즈니스 로직 (Interactions & MCP)
==================================================
1) [로그인 게이트]: 시연용 파트너 계정이 미리 입력되어 있으며, 버튼 클릭 시 로그인 창이 사라지며 메인 헤더 및 챗봇 탭이 활성화된다.
2) [시연용 질문 자동 채우기]: 챗봇 대화창 하단에 '💡 [시연 가이드] 샘플 질문 자동 입력' 클릭 요소를 제공하여, 클릭 시 일반 대중 투자자가 던질 법한 자연어 질문("대기업 말고 실속 챙기는 알짜 부품사 포트폴리오 짜줘")이 입력창에 자동 타이핑되도록 구현하라.
3) [Graph-RAG 실시간 대화 및 미리보기 빌드]:
   - 질문 전송 시 AI의 생각 중 연출 후 Graph-RAG 답변 말풍선이 생성된다.
   - 답변 직후, 좌측 대화 맥락을 기반으로 우측 미리보기 패널에 [대형주(삼성전자) 제외 ➔ 검사부품/IP설계 역할 분산 ➔ 체급 비중 정규화] 알고리즘이 적용된 ETF 비중(리노공업 40%, ISC 35%, 오픈엣지 25%)이 실시간으로 동적 렌더링된다.
4) [MCP 마켓플레이스 자동 상장 프로세스]:
   - 미리보기가 완성되면 [🚀 마켓플레이스에 일회성 열람 상품으로 즉시 상장하기] 버튼이 출현한다.
   - 버튼 클릭 시 MCP 프로토콜 통신 연출("원격 상장 등록 중...")이 실행된 후, 마켓플레이스 데이터베이스에 신규 카드(내가 만든 인덱스, 단건 열람료 1,500원)가 즉시 등록되며 **TAB 3(오픈 마켓플레이스) 화면으로 자동으로 강제 라우팅(탭 이동)** 처리된다.

=========================================
5. 구현 기술 스택 및 제약 조건
=========================================
- 100% 단일 HTML 파일로 작성해야 하며, 별도의 외부 라이브러리 설치 없이 순수 Vanilla HTML, CSS, JavaScript로 구현하라.
- 아이폰 사파리 모바일 와이드 및 PC 웹 브라우저 환경에서 화면 깨짐 없이 부드럽게 작동해야 한다.
- 위 조건에 부합하는 완벽한 웹 애플리케이션 코드를 바로 실행 가능하게 작성하라.
