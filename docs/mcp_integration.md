# MCP 연동 및 챗봇-커뮤니티 연계 방안 (MCP Integration)

본 문서는 AI가 포트폴리오를 직접 추천할 수 없는 법적 규제를 준수하면서도, **Model Context Protocol (MCP)**을 통해 챗봇(Tab 1)과 소셜 인베스팅 커뮤니티 피드(Tab 3) 간의 유기적인 연동을 극대화하는 아키텍처 및 구현 방안을 정의합니다.

---

## 1. 챗봇과 SNS 간의 연계 병목 현상 및 해결 프레임워크

### ① 문제 상황
- 자본시장법 규제에 따라 AI가 유저의 투자 성향을 기반으로 특정 ETF를 매수 추천(Recommendation)하는 것은 불법입니다.
- 이로 인해 챗봇과 소셜 피드 간의 흐름이 단절되어 서비스의 사용성이 크게 저하될 위험이 있습니다.

### ② 해결책: '자문(Advice)'에서 '검색 및 도구 대행(Search & Tool Execution)'으로의 전환
- AI가 자의적인 판단으로 상품을 추천하는 대신, 유저의 자연어 검색 의도를 파악하여 **MCP 도구(Tools)를 통해 커뮤니티 피드 DB를 Semantic 조회**하고 결과를 단순 중개/나열하게 만듭니다.
- 유저가 챗봇 대화 중 발견한 인사이트를 **MCP 액션(Actions)을 통해 커뮤니티 피드에 즉시 소셜 인덱스 리포트로 발행(Publish)**하도록 연동합니다.

---

## 2. MCP (Model Context Protocol) 연동 아키텍처

AI 모델(Gemini 등)이 MCP 규격에 맞추어 클라이언트(Frontend) 및 백엔드 데이터베이스와 통신하는 3가지 연동 루프입니다.

```
┌────────────────────────────────────────────────────────┐
│                     Client (Frontend)                  │
└───────────┬────────────────────────────────┬───────────┘
            │ (User Query / URI Resource)    │ (Action Trigger / Tab Redirect)
            ▼                                ▲
┌───────────────────────┐        ┌───────────┴───────────┐
│     AI Model (LLM)    ├───────>│  MCP Server (Backend) │
└───────────────────────┘ (Tool) └───────────┬───────────┘
                                             │ (Neo4j / SQL Query)
                                             ▼
                                 ┌───────────────────────┐
                                 │     Community Feed DB │
                                 └───────────────────────┘
```

### ① Search Loop (커뮤니티 피드 지수 Semantic 검색 도구)
- **MCP Tool**: `search_community_indices(keywords, min_yield)`
- **시나리오**:
  1. 유저가 챗봇에게 *"소셜 피드에 올라온 유저 인덱스 중에 한미반도체가 포함되거나 HBM3E 패키징 관련 포트폴리오가 있는지 찾아줘"*라고 요청합니다.
  2. AI는 **`search_community_indices`** 도구를 호출합니다.
  3. 백엔드는 커뮤니티 DB에서 관련 종목 및 태그가 일치하는 유저 ETF 리스트를 JSON으로 반환합니다.
  4. AI는 이를 가공하여 **"검색 조건에 맞는 커뮤니티 공유 지수 2건을 찾았습니다. 링크를 눌러 상세 비중과 요약을 확인해 보세요."**라고 출력하며 마켓플레이스 링크 카드 형태로 리스트를 렌더링합니다.
- **법적 정당성**: 상품의 자의적 추천이 아닌, 사용자의 명시적 지시에 의한 **'데이터베이스 검색 대행 서비스'**이므로 유사투자자문 규제 내에서 완전히 합법입니다.

### ② Publish Action Loop (MCP 원격 커뮤니티 피드 발행 및 라우팅 도구)
- **MCP Tool**: `publish_social_index(title, weights, description)`
- **시나리오**:
  1. 유저가 챗봇과의 대화(RAG 분석)를 통해 도출된 강소기업 비중 리스트에 만족하며, *"이 포트폴리오를 커뮤니티 피드에 게시해줘"*라고 지시합니다.
  2. AI는 **`publish_social_index`** 도구를 실행합니다.
  3. MCP 서버는 해당 데이터셋을 커뮤니티 데이터베이스에 공유 포스트로 등록 처리합니다.
  4. 등록 성공 이벤트 수신 시, 클라이언트(프론트엔드)는 MCP 원격 로딩 오버레이를 띄운 뒤 **소셜 피드(Tab 3)로 화면을 강제 라우팅(Tab Routing)**하고 새로 등록된 카드를 최상단에 하이라이트 노출합니다.
- **법적 정당성**: 사용자의 명시적인 입력(Publish 명령)을 받아 백엔드 DB 쓰기 작업을 수행하는 자동화 매커니즘이므로 규제 대상이 아닙니다.

### ③ Diagnosis Resource Loop (커뮤니티 데이터 컨텍스트 제공)
- **MCP Resource**: `index://<etf_id>` (예: `index://hbm3e-index`)
- **시나리오**:
  1. 유저가 소셜 피드에서 타인의 ETF 카드를 구경하던 중 **"이 ETF에 대해 챗봇에게 분석 질문하기"** 버튼을 클릭합니다.
  2. 서비스는 챗봇 탭(Tab 1)으로 이동하며, 해당 ETF의 식별자 리소스 URI(`index://hbm3e-index`)를 챗봇 컨텍스트에 주입합니다.
  3. AI는 해당 리소스 프로토콜을 통해 ETF 구성 종목(한미반도체, 에스티아이 등)과 비중을 읽어와 **"선택하신 HBM3E 패키징 인덱스의 종목 구성은 다음과 같으며, 최근 DART 공시 기준 공급망 관계는..."**이라며 즉각적인 팩트 진단을 수행합니다.
- **법적 정당성**: 특정 공유 자산(ETF)의 상세 구성 내용을 객관적으로 팩트체크하고 구조 분석을 제공하는 정보 분석 서비스이므로 합법입니다.

---

## 3. 개발팀 간 협업을 위한 MCP 규격 프로토콜

- **AI 백엔드 팀**:
  - LLM 모델이 쿼리 컨텍스트에 따라 `search_community_indices` 및 `publish_social_index` 도구를 정확한 타이밍에 바인딩 및 실행할 수 있도록 프롬프트 가이드라인(System Instructions)을 수립합니다.
- **커뮤니티 백엔드 팀**:
  - MCP 스펙(JSON-RPC 기반)에 호환되는 MCP 호스트 서버를 Node.js 또는 Python FastAPI로 기동하고, `/api/mcp` 엔드포인트를 열어 도구 실행 및 데이터 조회를 처리합니다.
- **프론트엔드 팀**:
  - MCP 도구 실행 결과로 발행되는 라우팅 및 팝업 이벤트를 가로채어, 로딩 스피너 작동 및 커뮤니티 피드 탭 자동 이동과 같은 동적 UX 연동을 처리합니다.
